R Base

The R distribution comes with several base packages, which are maintained by the R Core Team in the same Subversion tree with R: Upstream of R base packages. An overview of these packages is available at R FAQ.

Some of these base packages are attached at R initiation: base, methods, datasets, utils, grDevices, graphics, stats; while others are not: grid, tools, tcltk, splines, stats4, compiler, parallel, profile, translations. (Start a new R session and run grep('package:', search(), value = TRUE).)

The base, utils, stats packages are particularly important in mastering R. This article lists functions in these packages that constitute a good working vocabulary of base R. This article also covers some example data in datasets and functions in graphics.

Note: “(pure)” denotes pure functions, i.e. a function that always returns the same result on the same input and does not cause any semantically observable side effect.

Date-time format string: (see ?strptime for details) - Default to “%Y-%m-%d %H:%M:%S” for date-times and “%Y-%m-%d” for dates; - Conversion specifications shall conform to POSIX 1003.1-2008 standard, but implementations are platform-specific: - %Y, year with century (0:9999); - %m, month as decimal number (01-12); - %d, day of the month as decimal number (01-31); - %H, hours as decimal number (00-23); - %M, minute as decimal number (00-59); - %S, second as integer (00-61), allowing for up to two leap-seconds; - Locale-specific conversions are used where appropriate and available, accessible with Sys.getlocale("LC_TIME") and Sys.setlocale("LC_TIME", "en_US.UTF-8"). - %b, abbreviated month name in the current locale on this platform; - %p, AM/PM indicator in the locale, used in conjunction with %I and not with %H;

Functional Programming

Expression: (non-standard evaluation)

  • quote(), returns its argument (“call”);
  • call(), expression() create a ‘call’/‘expression’ object (including math symbols).
  • substitute(), returns the parse tree (modified expression) for the unevaluated expression: name - value; promise - expression; ‘…’ - expression; otherwise behaves like quote().
  • deparse(), turn unevaluated expressions into character strings.
  • eval(), evaluate an unevaluated expression in a specified environment.
  • all.names(), all.vars(), extract all the names/variables in an expression or call.

There’s a general tension between functions that are designed for interactive use (using substitute() to reduce typing) and functions that are safe to program with (called from another function). Escape hatch of a function using non-standard evaluation is its alternative version that takes already quoted expressions and uses standard evaluation.

Function as object:

  • formals(), body(), access the formal arguments and body of a function.
  • environment(), access the enclosure of a function (“closure”).
  • match.fun(), extract a function specified by name;

Functions as keywords:

  • function() {}, define a function.
  • missing(), test unaltered name.
  • force(), force evaluation of a formal argument: function(x) x.
  • return(), explicitly return values.
  • on.exit(), function exit code, useful for undoing side effects.

Functionals:

  • replicate(), repeatedly evaluate an expression.
  • apply(), apply a function to margins of a matrix/array: 1, rows; 2, columns; 3…;
  • lapply(), sapply() (recommended only for interactive use), vapply(), apply a function to a list (and simplify) or a vector.
  • tapply(), apply a function to vector grouped by types (a combination of split() and sapply()).

  • Reduce(), reduces a vector to a single value by recursively calling a two-argument function.
  • Filter(), selects only elements that match a predicate.
  • do.call(), apply a list of arguments to a function.

Function operators:

  • Vectorize(), parallelize a function with respect to the arguments specified.
  • Negate(), negate a predicate function.

Debug:

  • browser(), inspect the current environment interactively: n, execute the next command; s, step into the next function; f, finish the current loop or function; c, continue execution normally; Q, stop the function and return to the console.
  • debug(), debugonce(), debug a function.
